# decisions/

Registro de decisiones tecnicas y de arquitectura (ADRs).

## Formato de archivo
`YYYY-MM-DD-titulo-corto.md`

## Estructura de cada decision
```
# Titulo
**Fecha:** YYYY-MM-DD
**Estado:** Propuesta | Aceptada | Rechazada | Deprecada

## Contexto
## Decision
## Consecuencias
```

Claude consulta estos archivos antes de proponer cambios arquitecturales para no contradecir decisiones ya tomadas.
